# セーリング マークレイヤー アプリ — 引き継ぎ書類

最終更新: 2026-06-18

セーリングのコース設計＋現在地トラッキング用Webアプリの引き継ぎ資料です。
この1ファイルを読めば、別の人・別アカウントが開発を継続できます。

---

## 1. これは何か

スマホのブラウザで動く、セーリング用の単一ページWebアプリ（`index.html` 1枚）。

- **レースモード**: 本船（committee boat）の座標を基準に、トラペゾイド/トライアングル/上下(L-R)/スタートラインなどのコースを精密設計。各マークの緯度経度を計算し、LINEやテキストで共有できる。
- **練習モード**: 地図を見ながら「現在地→上マーク/下マーク/その他」をワンタップで配置。風向を入れると基準マークを通る **風軸ライン（実線）＋直交ライン（破線）** をライブ表示。
- **海面図オーバーレイ**: 相模湾レース海面図(PDF)を地図に正しい座標で重ねて表示。ON/OFF・透過調整・他の海面図の追加が可能。
- **GPS**: 現在地から最終マークまでの距離・方位をリアルタイム表示。

## 3. 技術構成（外部ライブラリはCDN）

- **Leaflet 1.9.4** — 地図ライブラリ（OpenStreetMapタイルを表示）
- **pdf.js 3.11.174** — PDFをブラウザ上で画像(PNG)に変換し、海面図として地図に重ねる
- 地図タイル: OpenStreetMap（無料・キー不要）
- フレームワーク無し。ビルド不要。`index.html` を開けば動く。
- データ保存: 追加した海面図は **localStorage**（その端末のブラウザ内）に保存。

レイヤーの重なり（下から上）:
```
④ マーカー・風軸/直交ライン
③ GPS現在地（青い点）
② 海面図（imageOverlay）
① OpenStreetMap（背景地図）
```

## 4. 別アカウントへの引き継ぎ手順

GitHubリポジトリを新しいアカウントに移す方法は3通り。**おすすめは A（譲渡）**。

### A. リポジトリ譲渡（Transfer）— 元アカウントから完全に移す
1. 旧アカウントで https://github.com/mitsuru-Tanaka/sailing-mark-layer/settings を開く
2. 一番下「Danger Zone」→ **Transfer ownership** → 新アカウント名を入力
3. 新アカウントで承認
4. 新アカウントの Settings → Pages で「Branch: main / root」を再設定
5. 新しい公開URLは `https://（新ユーザー名）.github.io/sailing-mark-layer/` になる
6. URLが変わるので、利用者に新URLを再共有する

### B. 新アカウントで作り直す（旧を残したい場合）
```bash
# 旧リポジトリをローカルに取得
git clone https://github.com/mitsuru-Tanaka/sailing-mark-layer.git
cd sailing-mark-layer
# 新アカウントで空リポジトリ sailing-mark-layer を作成しておき、リモートを差し替え
git remote set-url origin https://github.com/（新ユーザー名）/sailing-mark-layer.git
git push -u origin main
git push origin --tags
```
その後、新アカウントの Settings → Pages で main/root を有効化。

### C. 共同編集者として招待（アカウントは旧のまま、二人で編集）
旧アカウントの Settings → Collaborators → 新アカウントを招待。
URLは旧のまま。手早いが「別アカウントに移す」ではなく「二人で使う」形。

> 補足: 追加した海面図(localStorage保存分)は **端末ごと** なので引き継がれません。
> 全員で共有したい海面図がある場合は、その画像と座標(bounds)をコードに直接登録します（後述）。

## 5. 更新（デプロイ）の方法

`index.html` を編集 → コミット → push するだけで、1分ほどで本番URLに反映されます。

```bash
cd ~/Desktop/sailing-mark-layer
git add index.html
git commit -m "変更内容の説明"
git push origin main
```

GitHub CLI(`gh`)で認証済みなら追加設定は不要。未認証なら `gh auth login`。

## 6. 海面図の座標について（最重要ノウハウ）

海面図PDFは「ただの絵」で位置情報を持たないため、**四隅の緯度経度を割り出して** `imageOverlay` で地図に貼っています。

`index.html` 内のこの定数が相模湾海面図の四隅（南西〜北東）:
```js
const CHART_BOUNDS=[[35.193175,139.452344],[35.309259,139.655033]];
//                  [[south, west      ],[north,    east     ]]
```

### 座標の出し方（新しい海面図をコードに固定登録する場合）
1. PDFを高解像度画像にする（PyMuPDF等）
2. 経緯度の目盛り（例: 139°28′、35°18′）が画像の何ピクセル目かを読む
3. 「ピクセル↔緯度経度」の一次式を作り、画像四隅(0,0)と(W,H)の座標を逆算
4. その値を `CHART_BOUNDS` に入れる

### ハマりポイント（過去の失敗）
- 相模湾海面図は **経度が1.0分間隔・緯度が0.5分間隔**。最初これを取り違えて経度を0.5分と読み、東西が2倍に伸びてズレた → 1分に直して解決。
- 検算: 経度1分=277px、緯度0.5分=170px(→1分340px)、比 1.235 ≈ 1/cos(35.3°)=1.225 でメルカトル投影と一致。
- ブラウザ（特にPCのChrome/Safari）は**キャッシュが強い**。更新が見えない時は必ず **ハードリロード（Cmd+Shift+R）**。

## 8. 未解決の課題（引き継ぎ時点 / 2026-06-18）

1. **海面図のズレ（要対応）**: PCの大画面で見ると相模湾海面図が実際の海岸線と微妙にズレているとの報告あり。`CHART_BOUNDS` の微調整が必要。ズレの方向と量（例「北に1km」「東西に間延び」）を確認してから、boundsを平行移動／スケール調整する。Web Mercator投影の非線形分や、PDF四隅の外挿誤差が原因の可能性。
2. **海面図の自由変形**: 現在の位置合わせは「移動・拡大縮小」のみ（回転・台形補正なし）。傾いた図を貼りたい場合は `Leaflet.ImageOverlay.Rotated` プラグイン等が必要。
3. **追加海面図の共有**: localStorage保存のため端末間で共有されない。定番図はコードに固定登録する運用。
4. **海面図の削除UI**: 追加した海面図を選択して消すボタンは未実装（localStorageクリアでリセットは可能）。

## 9. 開発時の注意・ノウハウ

- **このMac環境ではターミナル(Bash)の標準出力が乱れる**ことがある（過去の出力が重複表示され grep/cat の結果が信用できない、ファイルが破損する事故も発生）。対策:
  - コマンド結果は `> /tmp/out.txt` に書いて、エディタ/Readで読む
  - ファイル全体を作り直す時は、シェルの `cat <<EOF` ではなくエディタで直接全文を書く
  - 編集後は必ずブラウザのコンソールでエラーが出ていないか確認
- **必ずバックアップを取ってから大改修する**: `git tag バックアップ名` ＋ `index.html` のファイルコピー。
- `.gitignore` に `index_backup_*.html` を登録済み（バックアップHTMLは公開されない）。
